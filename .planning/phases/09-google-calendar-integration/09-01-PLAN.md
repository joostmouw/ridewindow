**Maak lib/services/calendar_service.dart aan** als een plain Dart class (geen Riverpod provider, geen singleton pattern — directe instantiatie per tik):

De class heet CalendarService. Import: google_sign_in/google_sign_in.dart, extension_google_sign_in_as_googleapis_auth/extension_google_sign_in_as_googleapis_auth.dart, googleapis/calendar/v3.dart, ridewindow/domain/models/ride_slot.dart, ridewindow/domain/models/hourly_forecast.dart.

Maak een privé static final _googleSignIn instantie aan als GoogleSignIn met scopes: [CalendarApi.calendarEventsScope]. NIET aanmaken in main(), NIET in een provider constructor — uitsluitend hier als lazy static field (CAL-02: lazy OAuth).

Schrijf de publieke methode addRideSlotToCalendar met signatuur:
  Future<void> addRideSlotToCalendar(RideSlot slot, List<HourlyForecast> forecasts)

Stappen in de methode (per CAL-04):
1. Roep await _googleSignIn.signIn() aan. Als het resultaat null is (gebruiker annuleerde), gooi een Exception('Aanmelden geannuleerd').
2. Haal authenticatedClient op via: final client = await _googleSignIn.authenticatedClient()
3. Als client null is, gooi een Exception('OAuth client niet beschikbaar').
4. Maak CalendarApi(client) aan.
5. Bouw de weersamenvatting (CAL-03) als een String. Gebruik de forecasts lijst:
   - Gemiddelde temperatuur: bereken avg van temperatureC waarden die niet null zijn, afronden op geheel getal, format als "~{n}°C"
   - Neerslag: som van precipitationMm. Als 0 of leeg: "droog". Anders: "{n}mm"
   - Wind: gemiddelde windspeedKmh, afronden, format als "{n}km/u wind"
   - Combineer met komma: "~18°C, droog, 12km/u wind". Als forecasts leeg is, gebruik "Geen weerdata beschikbaar".
6. Stel de event title in als: "Fietsrit ${_fmtTime(slot.start)}–${_fmtTime(slot.end)}"
   Waarbij _fmtTime een privé hulpmethode is: "${dt.hour.toString().padLeft(2,'0')}:${dt.minute.toString().padLeft(2,'0')}"
7. Maak een Event object aan (uit googleapis/calendar/v3.dart) met:
   - summary: de event title
   - description: de weersamenvatting
   - start: EventDateTime(dateTime: slot.start, timeZone: 'Europe/Amsterdam')
   - end: EventDateTime(dateTime: slot.end, timeZone: 'Europe/Amsterdam')
8. Roep await calendarApi.events.insert(event, 'primary') aan (CAL-01, CAL-05).
9. Sluit de client af: client.close().

Omhul stap 2 t/m 9 in try-finally zodat client.close() altijd wordt aangeroepen.

## Vertrouwensgrenzen

| Grens | Beschrijving |
|-------|-------------|
| Gebruiker → GoogleSignIn | OAuth-flow verloopt via Google Sign-In SDK; tokens verlaten het apparaat naar Google |
| CalendarApi → Google Calendar | HTTPS API-aanroepen met OAuth-token; event-data wordt verstuurd naar Google |
| RideDetailScreen → CalendarService | Lokale methode-aanroep; geen data-overdracht tenzij gebruiker actief tikt |

## STRIDE Bedreigingsregister

| Bedreiging-ID | Categorie | Component | Dispositie | Mitigatieplan |
|--------------|-----------|-----------|------------|---------------|
| T-09-01-01 | Information Disclosure | GoogleSignIn OAuth-token | mitigate | Token wordt beheerd door Google Sign-In SDK (niet handmatig opgeslagen); client.close() in finally-blok zorgt voor opruimen |
| T-09-01-02 | Elevation of Privilege | CalendarApi scope | mitigate | Uitsluitend CalendarApi.calendarEventsScope — geen volledige calendar-toegang (per CAL-04) |
| T-09-01-03 | Information Disclosure | Weerdata in event description | accept | Weerdata is niet-persoonlijk (temperatuur, wind); gebruiker triggert zelf het aanmaken (PERS-04: geen data zonder expliciete toestemming) |
| T-09-01-04 | Spoofing | OAuth consent | mitigate | Google Sign-In SDK toont standaard Google consent screen — niet te omzeilen door de app |
| T-09-01-SC | Tampering | pub installs (google_sign_in etc.) | mitigate | Alle drie pakketten zijn gepubliceerd door flutter.dev / google.dev (vermeld in CLAUDE.md stack-tabel); uitgeververificatie voltooid |
