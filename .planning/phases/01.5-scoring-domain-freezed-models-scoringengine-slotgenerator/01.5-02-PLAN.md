Class: ScoringEngine

Method signature:
  HourlyScore score(HourlyForecast forecast, WeatherTolerances tolerances)

Sub-score computation rules (each sub-score is 0–100):

TEMPERATURE SUB-SCORE (_temperatureScore):
- If forecast.temperatureC is null → return 50.0 (uncertain)
- Ideal range: [tolerances.tempMinIdealC, tolerances.tempMaxIdealC]
- Score 100 when temperatureC is within the ideal range
- Below tempMinIdealC: score decreases linearly from 100 at tempMinIdealC to 0 at (tempMinIdealC - 20.0). Clamp to 0.
- Above tempMaxIdealC: score decreases linearly from 100 at tempMaxIdealC to 0 at (tempMaxIdealC + 20.0). Clamp to 0.
- Use apparent_temperature (apparentTemperatureC) as a secondary signal only if it is non-null: take the minimum of the temperature score and the apparent-temperature score (so feels-like extremes count against the score). If apparentTemperatureC is null, use temperatureC score alone.

RAIN SUB-SCORE (_rainScore):
- If forecast.precipitationMm is null → return 50.0 (uncertain)
- Score 100 when precipitationMm <= tolerances.rainMaxIdealMm
- Score decreases linearly from 100 at rainMaxIdealMm to 0 at (tolerances.rainMaxIdealMm + 5.0). Clamp to 0.
- precipitationProbability is not scored separately — it is only used in Phase 2+ forecasting context.

WIND SUB-SCORE (_windScore):
- If forecast.windspeedKmh is null → return 50.0 (uncertain)
- Score 100 when windspeedKmh <= tolerances.windMaxIdealKmh
- Score decreases linearly from 100 at windMaxIdealKmh to 0 at (tolerances.windMaxIdealKmh + 40.0). Clamp to 0.

OVERALL SCORE (D-14 formula):
  overall = 0.6 * min(t, r, w) + 0.4 * (t + r + w) / 3.0
  Clamp overall to [0.0, 100.0].

Return HourlyScore(
  overall: overall,
  temperatureScore: temperatureScore,
  rainScore: rainScore,
  windScore: windScore,
  time: forecast.time,
)

All helper computation should be in private methods _temperatureScore, _rainScore, _windScore for testability of the overall path. Each private method takes the relevant forecast field and the relevant tolerance field(s) as parameters (not the whole forecast/tolerances objects), making the null-handling path explicit and visible.

Add a top-level doc comment on the class:
/// Pure-Dart scoring engine for cyclist weather windows.
/// Null weather inputs clamp to 50/100 ("uncertain") per SCOR-04.
/// Overall formula: overall = 0.6·min(t,r,w) + 0.4·mean(t,r,w) per D-14.
