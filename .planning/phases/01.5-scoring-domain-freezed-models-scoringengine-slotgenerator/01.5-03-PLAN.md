Class: SlotGenerator

Method signature:
  List<RideSlot> generate(List<HourlyScore> scores, {required List<int> allowedDurations})

Where allowedDurations is a list of desired slot lengths in hours (e.g. [2, 3, 4, 5]).

Algorithm:
1. The input list is assumed to be sorted by time, one entry per hour.
2. For each allowed duration d in allowedDurations:
   a. Slide a window of d consecutive hours across the scores list.
   b. For each window position i (0-indexed), the window covers scores[i..i+d-1] (inclusive, d hours).
   c. Compute overallScore = average of the d hourly overall scores in the window.
   d. Compute tier = rideTierFromScore(overallScore).
   e. start = scores[i].time (inclusive)
   f. end = scores[i + d - 1].time + Duration(hours: 1) (exclusive — one hour past the last scored hour, implementing [start, end))
   g. Create RideSlot(start: start, end: end, overallScore: overallScore, tier: tier, hours: scores.sublist(i, i + d))
3. Collect all generated slots (across all durations and all window positions) into one flat list.
4. Return the list sorted by start time ascending, then by overallScore descending (so within the same start hour, highest-scoring duration comes first).
5. Duplicate slots with identical start/end pairs are allowed — the filter and display layer will deduplicate if needed.

Edge cases:
- If scores.length < d for a given duration, skip that duration (no window fits).
- If scores is empty, return [].

Add a class doc comment:
/// Generates ride slots of the given durations from a list of hourly scores.
/// Slot boundaries use the exclusive-end convention [start, end):
/// a 2h slot starting at 09:00 covers 09:00–11:00 with end == 11:00.
</action>

Class: AvailabilityFilter

Method 1 — filter out blocked slots:
  List<RideSlot> removeBlocked(List<RideSlot> slots, Set<DateTime> blockedHours)

Where blockedHours is a set of DateTime values each representing the start of a blocked hour (e.g. DateTime(2025,7,1,9) means the 09:00–10:00 hour is blocked).

A slot overlaps a blocked hour if any hour within [slot.start, slot.end) is in blockedHours.
Implementation: for each slot, iterate the slot's hours (slot.start, slot.start + 1h, ..., slot.end - 1h); if any of those DateTimes is in blockedHours, exclude the slot.

Method 2 — filter out Poor-tier slots (hidden per SLOT-04):
  List<RideSlot> removeHiddenPoor(List<RideSlot> slots)

Returns all slots where the tier is NOT Poor. Use a pattern match: `slot.tier is! Poor`.

Method 3 — apply both filters:
  List<RideSlot> apply(List<RideSlot> slots, Set<DateTime> blockedHours)
  => removeHiddenPoor(removeBlocked(slots, blockedHours))

Add a class doc comment:
/// Filters ride slots by availability and quality tier.
/// Poor-tier slots (score < 50) are hidden from the UI per SLOT-04.
/// Blocked hours use the same [start, end) convention as RideSlot.
</action>
