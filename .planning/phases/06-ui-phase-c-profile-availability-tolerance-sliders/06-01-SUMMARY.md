## Decisions Made

- **AsyncValue.value instead of .valueOrNull**: Riverpod 3.x `AsyncValue` exposes `.value` (returns `T?`) not `.valueOrNull`. The PATTERNS.md showed `valueOrNull` but that does not exist in the installed version (3.2.1). Switched to `.value` — analyzer confirmed no errors.
- **Unused field suppression**: The four `late double` fields (`_tempMin`, `_tempMax`, `_rainMax`, `_windMax`) are intentionally unused in Wave 1. Added `// ignore: unused_field` comments to suppress analyzer warnings while preserving the skeleton structure for Wave 2.
