## Phase Goal

**As a** developer, **I want to** build a signed release AAB and verify it on a physical device, **so that** I can confirm the app works in release mode before submitting to Play Console.

<objective>
This plan runs `flutter pub get` to pull url_launcher, then builds the signed release AAB and APK using obfuscation flags, sideloads the APK to a physical Android device, and performs a manual smoke test of all key features.

Purpose: REL-05 explicitly requires sideload smoke testing on a physical device before any Play Console upload. Debug symbols must be backed up because obfuscated crash reports from testers will be unreadable without them.
Output: Signed app-release.aab ready for Play Console upload; smoke test confirmed on physical device.
</objective>

       (Clears cached build artifacts that might retain old debug signing config)

         --obfuscate \

    4. flutter build apk --release \
         --obfuscate \
         --split-debug-info=build/app/outputs/symbols
       (Produces signed release APK at build/app/outputs/flutter-apk/app-release.apk)

    If any build step fails:
    - "Keystore not found" → verify ~/upload-keystore.jks exists and android/key.properties storeFile path matches
    - "License for package Android SDK Build-Tools not accepted" → re-run `flutter doctor --android-licenses`
    - "Gradle build failed" → read the full error; the most common cause is a missing Android SDK component (run sdkmanager to install)
    - Do NOT proceed to Task 2 if builds fail; report the error output

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| release APK → device | Sideload bypasses Play Protect — only the developer's device, not testers' |
| debug symbols → backup | Symbols expose app structure; store alongside keystore in password manager |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-05 | Tampering | Debug-signed AAB uploaded to Play Console | mitigate | Task 1 acceptance criteria verify build output line does not say "debug"; `flutter clean` clears cached debug signing before build |
| T-10-06 | Information Disclosure | Symbol files lost after flutter clean | mitigate | Task 2 Step 5 explicitly backs up symbols before accepting smoke test; symbols go to password manager entry |
| T-10-07 | Denial of Service | versionCode not incremented on future builds | accept | Low risk for v1 (first upload); future releases: increment pubspec.yaml +N before each Play Console upload |
</threat_model>
