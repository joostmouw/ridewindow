## Phase Goal

**As a** developer, **I want to** upload RideWindow to the Play Console Internal testing track and share the opt-in link with cyclist friends, **so that** real users outside my household can install and use the app.

<objective>
This plan completes the Play Console setup required before uploading the AAB: minimum store listing (name, icon, description, screenshots), content rating, Data Safety form (REL-04), and then the actual upload and Internal testing track publish (REL-06). Everything in this plan is manual Play Console web UI work — no Dart code changes.

Purpose: Without a minimum store listing, Play Console blocks the AAB upload. Without the Data Safety form, the app cannot proceed to Closed/Production tracks later. REL-06 is the final deliverable of Phase 10.
Output: Opt-in URL distributed to 10–20 cyclist testers; at least one tester has installed and opened the app.
</objective>

    Step 1: Go to play.google.com/console → Create app
      - App name: RideWindow
      - Default language: Dutch (nl-NL) or English
      - App or game: App
      - Free or paid: Free
      - Accept Developer Program Policies

    Step 2: Complete Minimum Store Listing (Main store listing):
      - App name: RideWindow
      - Short description (max 80 chars): "Vind je beste fietsmoment — weerscore + agenda in één blik"
      - Full description (max 4000 chars): Write 2–4 paragraphs describing what RideWindow does (cyclist weather scoring, availability calendar, ride slot ranking, notifications, optional calendar export)
      - App icon: Upload a 512×512 PNG. If no custom icon exists, create a simple one (e.g. bicycle wheel + weather icon, or use any image editor). This is required; the Flutter default blue diamond will work for internal testing.
      - Feature graphic: Upload a 1024×500 PNG (can be a simple screenshot-style banner).
      - Screenshots: Upload at least 2 Android phone screenshots (take them from the sideloaded app using adb: `adb exec-out screencap -p > screenshot.png`)

    Step 3: Set Privacy policy URL:
      In the store listing: paste the GitHub Pages URL from Plan 03 (e.g. https://joostmouw.github.io/ridewindow-privacy/)

    Step 4: Content rating:
      - Play Console → Policy → App content → Content rating
      - Complete the IARC questionnaire
      - Category will be "Utilities" or "Tools"
      - No violence, no user-generated content, no location sharing with others
      - Save and confirm rating

    Step 5: Data Safety form (REL-04):
      Play Console → Policy → App content → Data Safety
      Declare the following:
      - Does your app collect or share any of the required user data types? YES
      - Location → Precise location: COLLECTED. Purpose: App functionality. Is this data required? Yes (app cannot function without it for core feature). Encrypted in transit? Yes. Can user request deletion? Yes (by revoking permission in Android settings).
      - Personal info → User's email address / Google account: This is used ephemerally via Google Sign-In for Calendar OAuth when the user explicitly taps "Add to calendar". Declare: Collected, purpose = App functionality, optional, user-initiated, not shared with third parties except for the purpose of Calendar event creation. Mark data as NOT stored on your servers (it stays on device).
      - Submit the form.

    Step 6: Verify the Play Console app dashboard shows no blocking errors (red X marks) before proceeding to Task 2.
